# Code Style & Conventions

## TypeScript/React
- **File naming**: PascalCase for components (e.g., `App.tsx`), camelCase for utilities
- **Component style**: Functional components with hooks
- **Type annotations**: Use TypeScript types explicitly
- **Imports**: ES6 imports, group by external packages first, then internal modules

## Styling
- Use Tailwind CSS utility classes
- Responsive design: mobile-first approach
- Color scheme: Sky/Blue gradient theme (sky-50, sky-600, blue-700, etc.)
- Spacing: Consistent use of Tailwind spacing scale

## Component Structure
- Export component as default
- Props interface defined before component
- Destructure props in parameters
- Use semantic HTML elements

## Design Patterns
- Single-file components when appropriate
- Separation of concerns: components, styles, logic
- Use of motion/animation for better UX
- Mobile-responsive design mandatory
